## Earlier version summary

The earlier version focused on lock-free correctness and benchmarking:

1. `TaggedStack`
- Packed `(index, version)` in one 64-bit atomic head.
- Demonstrated ABA protection via version increments on CAS.
2. `HazardPointerStack`
- Published per-thread hazard pointers before reading/removing nodes.
- Retired nodes and reclaimed only when no hazard references remained.
3. Benchmark intent
- Compare push/pop throughput under contention.
- Explain memory ordering and reclamation overhead trade-offs.

## New version summary (current)

The new version keeps the same core algorithms and adds explicit finance mapping in output and narrative:

1. Problem 102 (Arbitrage)
- Tagged CAS is explained as protection against stale execution paths during fast updates.
2. Problem 105 (Dividends / Price resets)
- Hazard-pointer retire/scan is shown as the safe mechanism for replacing or deleting nodes while readers are active.
3. Runtime output changes
- Added direct labeling in console output for arbitrage and dividend-reset interpretation.
- Added a reset-style write path note to show safe update semantics under reclamation.

## What did not change

1. Core lock-free data structures remain the same.
2. Stress benchmark methodology remains the same.
3. Throughput interpretation remains the same: hazard pointers usually add pop-path overhead but provide lifetime safety.
